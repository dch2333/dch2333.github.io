Chrome浏览器扩展开发

  • CSS
  • HTML
  • JS

Chrome自带的翻译有的时候会出现无法翻译的情况，我也需要一个使翻译的元素停留在页面上的功能，所以就萌生了写一个自己的划词翻译的想法。

扩展开发教程参考了这篇文章

功能设计

首先确定需求

  • 我不需要知道该单词的详细信息，只要一个大概的意思就好
  • 翻译出的单词或句子要在页面驻留，方便我再次查看
  • 还要有一个方便使用的主动输入的翻译工具，省的我再打开一个网页再搜索翻译工具

配置文件-manifest.json

这个文件用来告诉浏览器关于扩展的一些基本信息，包括名称、版本、所需权限等。

翻译接口

使用了百度开放平台的翻译接口，每月200万字符以内免费。

这是官方文档给出的例子：

例：将apple从英文翻译成中文：

请求参数：

q=apple

from=en

to=zh

appid=2015063000000001

salt=1435660288

平台分配的密钥: 12345678

生成sign：

>拼接字符串1

拼接appid=2015063000000001+q=apple+salt=1435660288+密钥=12345678

得到字符串1 =2015063000000001apple143566028812345678

>计算签名sign（对字符串1做md5加密，注意计算md5之前，串1必须为UTF-8编码）

sign=md5(2015063000000001apple143566028812345678)

sign=f89f9594663708c1605f3d736d01d2d4

完整请求为：

http://api.fanyi.baidu.com/api/trans/vip/translate?q=apple&from=en&to=zh&appid=2015063000000001&salt=1435660288&sign=f89f9594663708c1605f3d736d01d2d4

也可以使用POST方法传送需要的参数。

BackGround.js

选中文本后右击，在菜单中选择扩展选项，使用params.selectionText获取文本值后调用百度API进行翻译，将翻译后的结果使用chrome.tabs.sendMessage发送给当前页面的content-script

注意消息传递必须在ajax请求的success中，因为jq的ajax请求默认是异步的，如果是发送ajax的返回值的话只会发送一个undefined过去。

content-script

content-script.js负责接收BackGround发来的消息并将其显示在当前网页上。
